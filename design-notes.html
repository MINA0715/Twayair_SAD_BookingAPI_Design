📐 Design Notes v4.0.6.8

t’wayair Booking Application Programming Interface (Booking API) enables third-party applications to request information (such as availability, flight information, fare details) and create or modify reservations in the t’wayair sysetm.

t’wayair Booking API will be exposed as a Web Service and will be using XML/HTTP(S). The definition of the services will be available to the third-party applications using HTTP(S).

Component Diagram

Client Tier

The client tier consists of the OTAs that will use the t’wayair Booking API system to process their requests. The interactions are through XML/HTTP(S). The OTAs will be using a customized XML Request Response structure to interact with the t’wayair Booking API system.

Application Tier

The application tier will process the client OTAs requests using Spring container in a Java EE application server. The application processing involves activities like business module processing, Database access etc. The application tier is divided based on their functional responsibilities into five layers namely Services Layer, Business Layer, Data Layer, XSLT Layer and Core Layer. The core layer concentrates on having functionalities like Caching, and Logging which is used in any of the three layers mentioned.

External Tier

The external tier consists of the database, CRS system and the t’wayair Payment Gateway system. The databases are used to store referential data, route specific and agency code details. The CRS system provides APIs for Shopping and booking solutions using their exposed services. Payment system provides a payment gateway for all Booking operations.
